Правила наименования переменных

Есть несколько правил при определении переменных:

имена переменных не должны повторять уже встроенные в Python имена, например, print, str, in и т.д.;
имена переменных могут содержать только буквы, цифры и нижнее подчёркивание;
имена переменных не должны начинаться с цифры;
очень желательно, чтобы имена переменных выражали суть переменной и не были слишком длинными.

Строка, которая не содержит ни одного символа — не то же самое, что None, и всё ещё будет являться строкой. Чтобы узнать это наверняка, можно использовать функцию type():

Строку можно задать разными способами. Выше мы указывали её содержимое явно. Но, кроме этого, можно получить текст у пользователя через стандартный поток ввода — так по-умному называется интерфейс командной строки. Ещё его часто называют консолью. 

Давайте попробуем это сделать и сразу же выведем полученный текст обратно:

1) Сложение через оператор "+":

>>> print("This is the first string" + " and this is the second")

>>> print("This is the first string" + " " + "and this is the third")
This is the first string and this is the third

\n используется для обозначения переноса строки.

3) Если склеивание требуется только на выводе в консоль, а не для изменения содержания переменных, можно перечислить через запятую все объекты, которые мы хотим вывести, и передать их в функцию print().

Это, конечно, неправильно. Поэтому, чтобы корректно сравнивать числа типа float, нужно использовать переменную, которая задаст необходимую точность. Из-за конвенций математического анализа её принято называть epsilon.

>>> (1.1 + 2.2) - 3.3 < epsilon
